[GCP] Persistent Account Compromise via Unauthorized GCP Infrastructure

Credited As: Develop Aguascalientes

Vulnerability Type: Remote Code Execution (RCE) / Abuse

Program: Google Abuse VRP

Executive Summary

This report details a sophisticated, long-term, and persistent compromise of a Google Account, initiated by a trojanized Android application. The attacker used the granted permissions to create multiple unauthorized Google Cloud Projects, leveraging this infrastructure for large-scale malicious operations. The core of the attack lies in its stealth techniques: using APIs to perform actions and then immediately deleting the evidence from standard user interfaces. Critically, the attacker manipulated IAM permissions to lock the legitimate owner out of deleting these unauthorized projects, ensuring their persistence.

Key Findings

  • Unauthorized GCP Infrastructure: Five unauthorized Google Cloud Projects were created on the account, with the primary project identified as cedar-nebula-170k4.
  • Large-Scale Malicious Operations: The GCP dashboard revealed alerts related to high-level computing services, including Virtual Machines (VMs) and GKE workloads, proving large-scale resource abuse.
  • Stealth & Evasion Techniques: The attacker actively hid their tracks by deleting evidence (e.g., Google Group memberships, Google Tasks) from the UI, creating a direct conflict with the historical data summary in the Google Dashboard.
  • Advanced Persistence & Administrative Lockout: The "Delete" and "Shut Down" buttons in the Cloud Console were disabled. IAM policies were manipulated to deny the legitimate owner permission to view or manage the projects.
  • Initial Compromise Event: The earliest evidence of compromise was an outgoing SMS from the Android device containing a DCB_ASSOCIATION string, a behavior consistent with spyware hiding its communications.

Attack Scenario & Impact

The attacker, likely the distributor of a trojanized Android application, tricks a user into granting broad Google Account permissions. This grants the attacker a sophisticated, persistent, and nearly undetectable foothold within the victim's Google ecosystem, allowing for multiple levels of abuse:

  • Complete Hijacking of Cloud Infrastructure: The ability to create and control multiple Google Cloud Projects under the user's identity to hijack powerful computing resources for malicious purposes (e.g., crypto mining, botnets).
  • Administrative Lockout & Evasion: The power to manipulate IAM permissions to lock the legitimate account owner out of administering or deleting the malicious cloud projects, ensuring long-term persistence.
  • Potential for Full Data Exfiltration: With this level of API access, the attacker could potentially access and exfiltrate all of the user's sensitive data, including emails, contacts, and files.
